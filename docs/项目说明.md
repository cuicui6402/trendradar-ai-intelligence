## TrendRadar 项目说明

### 项目背景

技术迭代速度快，求职者和开发者经常面临三个问题：某项技术是否值得学习？简历项目该围绕哪个方向选题？团队技术选型有什么数据支撑？

TrendRadar 通过采集 GitHub 和 Hacker News 上的公开数据，计算趋势评分，生成面向求职和学习的分析报告，帮助回答这些问题。

### 项目解决的问题

- **信息分散**：GitHub 和 Hacker News 上的技术热度数据分散在不同页面，手动对比耗时
- **单一指标局限**：只看 Stars 或 Points 无法区分"历史热门"和"当前热门"，需要综合多维度指标
- **缺乏求职导向**：现有工具大多面向通用技术监控，没有针对求职选题和面试准备的输出

### 目标用户

- 正在准备求职的开发者（选题、面试准备）
- 需要做技术选型的工程师或技术负责人
- 关注技术趋势的学习者

### 核心功能

1. **多源数据采集**：同时调用 GitHub Search Repositories API 和 Algolia HN Search API，采集近期活跃数据
2. **多级 fallback 查询**：当精确搜索无结果时，自动放宽时间范围和关键词匹配策略，提高数据召回率
3. **趋势评分模型**：综合 Stars/Forks/Issues/Points/评论数/时间新鲜度，计算 0-100 加权评分
4. **相关性筛选**：报告中的 GitHub 代表项目经过相关性评分过滤，排除面试指南、算法题库等资料类项目
5. **AI 趋势报告 Agent**：双模式运行（无需 API Key 的模板模式 + 可选的 LLM 深度分析），生成 9 章节结构化报告
6. **SQLite 历史记录**：每次搜索自动保存，侧边栏展示搜索历史
7. **Markdown 报告导出**：报告自动保存为 Markdown 文件，数据库同步记录

### 数据来源

| 来源 | API | 数据内容 |
|------|-----|---------|
| GitHub | Search Repositories API | 仓库名、Stars、Forks、Issues、语言、创建/更新时间、描述 |
| Hacker News | Algolia HN Search API | 帖子标题、Points、评论数、作者、创建时间 |

两个 API 均为公开接口，无需 API Key 即可使用。可选配置 GitHub Token 提高速率限制。

### 系统流程

```
用户输入关键词
    ↓
GitHub Collector ──→ GitHub API ──→ 仓库数据
    ↓                                    ↓
HN Collector ────→ HN API ──────→ 帖子数据
    ↓                                    ↓
Scoring Service ──→ 归一化 + 加权评分 ──→ trend_score (0-100)
    ↓
Streamlit 前端 ──→ 表格 + 柱状图 + 排行榜
    ↓
TrendAgent ──→ 相关性筛选 + 模板/LLM ──→ 9 章节 Markdown 报告
    ↓
SQLite DB ──→ 搜索历史 + 评分结果 + 报告记录
```

### 技术栈

- **前端**：Streamlit
- **后端**：Python 3.11+
- **数据采集**：Requests
- **数据处理**：Pandas、NumPy
- **数据库**：SQLite + SQLAlchemy 2.x
- **可视化**：Plotly
- **报告**：Markdown 模板引擎 + OpenAI 兼容 API（可选）
- **测试**：pytest 风格自测脚本
