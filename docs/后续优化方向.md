## TrendRadar 后续优化方向

> 以下是已规划的优化方向，按优先级排列。每个方向独立，可以逐步实现。

---

### 数据源扩展

**接入国内技术社区**

目前 GitHub 和 HN 都是英文社区数据，对中文用户覆盖面有限。可以接入以下数据源：

- V2EX API：国内开发者社区，技术讨论活跃度较高
- 掘金：国内前端/后端/AI 方向的技术博客平台
- InfoQ 中文站：面向架构师和技术管理者的内容

接入方式与现有的 GitHub/HN 采集器类似：创建新的 Collector 类继承 BaseCollector，实现 `collect_df` 方法，在 scoring_service 中添加对应的评分函数。

**大厂开源组织监控**

追踪国内外大厂（字节、阿里、腾讯、Google、Microsoft 等）的开源组织页面，监控近期新建和高增长的仓库，作为技术趋势的早期信号。

### 功能增强

**定时任务与周报**

使用 APScheduler 或系统 cron 定时执行关键词监控任务，每周自动生成趋势周报。报告可以通过邮件或 IM 推送（如飞书/钉钉 webhook）。

**报告 HTML 导出**

目前只支持 Markdown 导出。可以增加 HTML 导出，包含图表渲染，方便直接在浏览器中查看和分享。实现方式：用 Jinja2 模板将 Markdown 转换为带样式的 HTML 页面。

**多关键词对比**

支持同时输入多个关键词（如"LangGraph vs CrewAI vs AutoGen"），在同一报告中对比不同技术的热度、项目数量和评分分布。

### 性能优化

**Redis 缓存**

热门关键词的搜索结果可以缓存到 Redis，设置 TTL（如 1 小时），避免重复请求 API。缓存键为 `keyword + days` 的组合。

**异步请求**

将 GitHub 和 HN 的采集改为 asyncio + aiohttp，实现真正的并行请求，减少用户等待时间。

### 部署与工程化

**Docker 部署**

编写 Dockerfile 和 docker-compose.yml，支持一键部署。包含 Streamlit 应用和 SQLite 数据卷挂载。

**FastAPI 接口层**

在 Streamlit 前端之外，提供 RESTful API 接口，支持其他系统（如移动端、飞书机器人）调用趋势分析能力。

**React 前端**

用 React + TypeScript 重写前端，提供更灵活的交互和更好的用户体验。后端通过 FastAPI 提供数据接口。

**CI/CD**

配置 GitHub Actions，在每次 push 时自动运行测试、lint 检查和构建验证。

### 模型优化

**评分权重学习**

用历史搜索数据（哪些技术后来确实变热了）做简单的回归分析，验证和调整评分权重。

**报告质量评估**

设计简单的报告质量指标（如覆盖率、数据引用准确率），用于衡量不同版本报告的质量变化。
